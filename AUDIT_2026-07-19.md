# Audit — v756–v766 (the mandatory post-stretch audit, David's law #6)

_Run 2026-07-19 after 4 deploys (v763–v766). Four parallel audit agents across the 12 AUDITS.md
lenses; EVERY finding was re-verified against the actual code by the lead before fixing (agent
claims were not trusted blind). Shipped as v767._

## Fixed (12 findings)

### HIGH
1. **Tour crash — shared `tourIndex` never reset when the task guide starts.**
   `Onboarding.tsx` — page tours and the 2-step task guide share `tourIndex`. Starting the task
   guide left the index at a page-tour value (e.g. 5); `TASK_TOUR[5]` is undefined → `step.route`
   deref → error boundary killed the tour. Fix: reset to 0 on fire. VERIFIED: fresh flow reveals
   the app, no crash.
2. **Moment-read upsell dead-ended on "Contact your Admin" with no capture.**
   `Planner.tsx` — a consumer has no "Admin"; the one user who tapped Subscribe got cold enterprise
   copy and no way to be told when it ships. Fix: the same `NotifyMeButton` waitlist as every other
   locked feature. VERIFIED (typecheck + the waitlist path was DB-verified in v765).
3. **My own v766 prosperity fix was incomplete + costly (found by the cost/cache agent).**
   The prosperity rule landed only in `GLANCE_TAIL`; the hero DAY read (`DAY_READ_TAIL`, the surface
   David actually saw) never got it, yet I bumped `day_read`'s cache salt anyway → a full paid
   regeneration of byte-identical output that STILL wouldn't name prosperity. Fix: added the rule to
   `DAY_READ_TAIL`, so the salt bump is now backed by a real change and both surfaces name a Sampat
   day. VERIFIED: `personalApex.tara="Sampat"` reaches the input on real Sampat days.

### MEDIUM
4. **`self-worth` — the banned 2nd-house framing — reached user prose** (`ProfectionYear.tsx`
   `HOUSE_GLOSS[2]`). Fix: "money, what you own, what you earn, your voice."
5. **Naked house number in a reading-voice lead** ("House 9 is lit up this year",
   `ProfectionYear.tsx`). Fix: "These are the areas of life in focus this year…" (the pills name them).
6. **Capture card: backdrop tap marked onboarding complete, stranding a chartless account.**
   `FirstRunWelcome.tsx` — an accidental backdrop tap ran `completeWelcome`, and the capture beat only
   renders while `!seen("welcome")`, so recovery was only the Profiles banner. Fix: no backdrop
   dismiss — an explicit choice (× or a footer button) is required. VERIFIED: card stays up on backdrop tap.
7. **Capture card showed two filled primaries** ("Save my birth details" + "Show me around"), and the
   tour button ran with no birth-saved check. Fix: while the birth form is open, "Show me around" is
   outline so "Save" is the lone primary. VERIFIED visually.
8. **Nudge arming race** (`OverlaySequencer.tsx`) — `nudgeArmed` was set before the timer, so a query
   refresh in the 1.8s window ran the cleanup (killed the timer) then re-ran the effect, which
   early-returned on the ref → the missing-location/bell nudge silently vanished. Fix: mark armed only
   when the timer fires; gate on `locationData.isSuccess`.

### LOW
9. **Nudge fired the bell instead of the location prompt while the location query was still loading**
   (`noLocation` was undefined) — fixed by the `isSuccess` gate above.
10. **Pinned "Keep this reading" used raw gold** (`#C9A84C`), invisible on the dark ruby/rose hero
    (the recurring gold-on-dark bug). Fix: label rides hero-ink; the filled BookmarkCheck icon carries
    the pinned state.
11. **`status.publicKey!` non-null assertion** (`MorningBellNudge.tsx`) could throw if configured
    without a key. Fix: guard → clean error state.
12. **`profiles.create` cap counted archived rows** — a cap-2 user who archived their 2nd profile was
    dead-ended. Fix: exclude `archivedAt` from the count.
