The five standing audits — every release, never leave the brief

David's doctrine (2026-07-22). These are the release gates the whole project answers to; the mechanical checklist that runs them in code lives in AUDITS.md (audits 1–14). Numbered as David wrote them.

7Narrative Fidelity

The astrology engine has already decided the meaning. The LLM is only allowed to voice it. Full 16-point spec: NARRATIVE_AUDIT.md (the engine→LLM contract, verbatim).

Before every release, verify:

  • Every narrative fact is fully resolved before the prompt.
  • The LLM never performs astrology.
  • One governing idea per reading.
  • One protagonist.
  • One emotional arc.
  • No competing narratives.
  • Remove true-but-unnecessary engine facts.
  • Endings distill rather than summarize.
  • Reflection questions arise naturally from the governing idea.

Failure = do not ship.

8Design Excellence

Every visible surface must feel intentional.

Audit every release for: typography · spacing · hierarchy · contrast · accessibility · animation · iconography · empty states · notifications · onboarding · loading · splash · paywall · dark mode · light mode.

Nothing ships because "it's good enough." It ships because nothing obvious remains.

9Revenue Integrity

Every release must prove:

  • no entitlement leaks
  • no subscription leaks
  • no purchase bypass
  • no accidental free access
  • no pricing inconsistency
  • no downgrade bugs
  • no restore failures

Every dollar should have one path. No hidden paths.

10Cognitive Load

Every screen answers immediately: Where am I? · What matters? · What happens next?

Every interaction removes work — never adds it. The user should never need to remember anything the interface can remember instead.

11The Velea Standard

Every change is measured against four questions:

  • Accuracy — Is it true?
  • Precision — Is this the clearest implementation possible?
  • Trust — Would this increase or decrease confidence in Velea?
  • Friction — Does this create unnecessary effort?

If any answer is "yes," fix the source before shipping.

The productivity Planner — neurodivergent-UX, greenlit 17 July

Evidence-based, from the market deck: neuroinclusive is universal design — the same choices that unblock an ADHD user unblock every tired, distracted, overloaded one. None of the five leaders (Notion, Todoist, TickTick, ClickUp, Motion) ship #1, #3 or #9 — open ground. These drop onto the Planner that already exists.

  1. One task / Focus Mode — hide all but the current action, one primary CTA. The app leans this way via soft-open + collapse; a dedicated mode is unbuilt.
  2. AI task-decomposition on capture — break a task into 3–7 micro-steps, first pre-selected; kills initiation paralysis at the moment it strikes. Shipped v710 per the record — confirm on a live look.
  3. Gentle overdue — overdue fades to neutral + one-tap "roll everything forward"; never a stack of red badges (shame → avoidance → churn). Shipped v708 per the record — confirm on a live look.
  4. Sensory control panel — the user controls animation, sound, colour-intensity, density; no autoplay, no unprompted motion. Needs your UX spec.
  5. Visual time-anchoring — every task shows an estimated duration; a shrinking-disc timer (not numerals) for the active one. Time-blindness accommodation, Motion's whole moat. Needs a duration schema script — yours to run.
  6. Zero-friction capture — voice-first, natural-language parse, capture from lock screen / share sheet, under two seconds. Never paywall capture.
  7. Save-and-resume by default — every flow pausable and resumable with visible progress and a one-line "you were here" re-orient.
  8. Predictable structure, opt-in novelty — navigation never moves; redesigns ship as opt-in themes (the hard no-jarring-redesign rule).
  9. Body-doubling & forgiving rewards — optional co-working presence; streaks that forgive gaps rather than reset to zero.
  10. Pomodoro focus sessions — your addition. A 25-min shrinking disc on a task → rest bell → 5–10 min break (a mini rest-gate, never guilt) → a different return bell. The two bells are your art. Parked; shares the duration script with #5. Trigger: "go pomodoro".
  11. Per-day task list — tap a day, see its tasks (added 22 Jul). Tapping a calendar day changes the reading but no task list follows it — the two task surfaces ("To Do", "Aligned for today") are today-locked. This would show the SELECTED day's tasks, tilted toward that day's mode (soft rank, exactly like the v929 Aligned fix: on-mode leads, the rest reachable below). A rough skeleton exists as dead code (matchedTasks in Planner.tsx, currently mode-HARD-filtered and rendered nowhere) — building this means wiring it up and softening it. You ruled "leave it for now."
