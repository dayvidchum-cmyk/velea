5 · Standing laws

Data is the source of truth. The engine locates; the model only voices. Accuracy outranks everything — UX and cosmetics included — in every triage.

Nothing is hardcoded unless the data that decides it is displayed where it is displayed, and what it says is accurate to the data. Yours, 21 July. A fixed string is legitimate only when the data that selected it is visible in the same place and the words are true to it — otherwise the app asserts something the reader cannot check and I cannot verify. It convicted a line the minute it was stated.

Fix the source, not the symptom. Name the root in one sentence, then fix the class, not the instance. Three times this run I fixed the instance and called the class closed; each time it came back.

A control that can fail is worth more than a test that passes. Three of my own tests were guarding nothing until I broke the code on purpose to check. Green tells you nothing on its own.

The instrument is wrong before the code is. Seven searches and counts this run gave a confident wrong answer before the right one — one of them claimed nine paid endpoints were ungated, and all nine were false. Every sweep now runs a control first.

Written is not deployed. Production sat frozen for fifteen versions while I wrote “deployed”. Nothing is deployed until the live site says its own version back.

Never ship a visual I have not looked at, and never auto-run a schema change.
