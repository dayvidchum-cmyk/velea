## Fixed

### Data-path (accuracy law — highest priority)
- **HIGH · The Why / Profection trigger sampled the wrong sky.** currentTransits built the
  input with NO dayLoc while every reading passes it → transit Moon anchored at noon-UTC vs
  the reading's viewer-local noon; ~6.5° drift at a 12h offset → the Moon (and natal-hit
  flags) could show in a different house than the reading voiced (~1 day in 5 for far-offset
  users). Now threads dayLoc (also shares the in-proc memo — no second ephemeris build).
- **HIGH · Crown/tara from Boston while the day-star was viewer-local.** input-builder's
  crownDay path called `majorityDayStarIdx(dateStr)` (Boston default) while personalDayForDate,
  getDayField and dayFilter were all viewer-located — the exact "Selective vs Restrained Build"
  split the v742 fix closed, surviving one call site up. Now dayLoc-seeded.

### Morning Bell
- **HIGH · double-ring + retry-hammer.** Non-atomic read→send→write dedupe, no reentrancy
  guard, prune only on 404/410. Fixed: claim-before-send (atomic conditional UPDATE stamps the
  date FIRST, then rings; 0 rows claimed = another tick/instance owns today) + tick reentrancy
  flag. Inverts every failure mode: a crash / hang / non-404 push failure now costs one missed
  bell, not a double-ring or 60-attempts/morning hammer (rotated VAPID keys → 403 forever).
- **HIGH · Off toggle couldn't silence other devices.** `status.subscribed` is per-user, so a
  laptop showed "On" but could only delete a bogus `"-"` row → snapped back. unsubscribe
  endpoint is now optional = clear ALL the user's devices.
- **MEDIUM · SSRF sink.** subscribe accepted any URL the server then POSTs to daily. Now
  requires https + a recognized push-service host.
- **MEDIUM · invalid tz rang at Boston-8am.** getTimezoneOffset swallows bad tz → Boston
  fallback. Now validated via Intl → skip (the header contract).
- **MEDIUM · deleteUserCascade** never cleared pushSubscriptions → stranded rows. Added.
- **MEDIUM · no TTL.** A morning line could arrive at 10pm / three at once after downtime.
  Now TTL 6h + topic (newer bell replaces undelivered) + 10s timeout (one hang can't stall).
- **MEDIUM · nudge mis-diagnosed transient errors** as "notifications off" and offered no
  retry; z-index 95 buried it under onboarding/date overlays. Fixed: separate retryable error
  state; z raised to 9998.
