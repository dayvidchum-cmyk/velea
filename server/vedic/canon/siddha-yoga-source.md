# SIDDHA YOGA — the grid I twice refused to build, found in a book already cited here

**Source:** B. V. Raman, *Muhurtha (Electional Astrology)*, **Chapter VI, "On Certain Special
Yogas", pp. 38–40.** Same volume `melana.json` and `amritaSiddhiYoga` already cite.

## The retraction

Twice I told David the Siddhi grid could not be built because **"I do not have a cited source for
them in this repo"** (recorded in `yoga-layers-doctrine.md`). He answered: *"muhurta yogas and
siddhi grids are definitely in the textbooks. i think you are making assumptions."*

He was right. The grid is in Chapter VI of a book sitting in his own source folder — **two pages
before the Amrita Siddhi verse I had already transcribed from that same chapter.** I had extracted
p.40 and never read p.38. The refusal to type a table from memory was correct; **not looking for the
table before declaring it absent was not.** That is the second time this run that a confident
negative of mine was really an unrun search.

## Raman's opening rule

> "When a certain week day coincides with a certain asterism and a certain thithi, it becomes
> specially auspicious for good work. In this chapter, I shall give a few such special combinations
> which go under the special distinction of **Siddha Yogas**."

## The grid, verbatim (OCR normalised only where marked)

| Day | Lunar days (tithi) | Constellations |
|---|---|---|
| **Sunday** | 1, 4, 6, 7, 12 | Pushya, Hastha, Uttara⚠, Uttarashadha, Mula, Sravana, Uttarabhadra |
| **Monday** | 2, 7, 12 | Rohini, Mrigasira, Punarvasu, Chitta, Sravana, Satabhisha, *Blwvishta*⚠, Purvabhadra |
| **Tuesday** | *(none given)* | Aswini, Mrigasira, Chitta, Anuradha, Moola, Uttara⚠, Dhanishta, Purvabhadra |
| **Wednesday** | Bhadra (2,7,12) & Jaya (3,8,13) | Rohini, Mrigasira, Ardra, Uttara⚠, Uttarashadha, *Animidha*⚠ |
| **Thursday** | 4, 5, 7, 9, 13, 14 | Makha, Pushya, Punarvasu, Swati, Purvashadha, Purvabhadra, Revati, Aswini |
| **Friday** | Nanda (1,6,11) & Bhadra (2,7,12) | Aswini, Bharani, Ardra, Uttara⚠, Chitta, Swati, Purvashadha, Revati |
| **Saturday** | Bhadra (2,7,12) & Riktha (4,9,14) | Swati, Rohini, Visakha, Anuradha, Dhanishta, Satabhisha |

**A second, separate list** — weekday × tithi only, no nakshatra required:

> "A Friday coinciding with **Nanda** (1st, 6th and 11th lunar days), Wednesday identical with
> **Bhadra** (2nd, 7th and 12th), Tuesday coinciding with **Jaya** (3rd, 8th and 13th), Saturday
> falling on a **Riktha** (4th, 9th and 14th), and Thursday falling on 5th, 10th or 15th (**Purna**)
> lunar days constitute Siddha Yoga."

## Raman's own calibration — which matches David's

> "The above special Yogas can be applied with advantage to important elections and if in addition
> to the general strength of the day due to a special yoga, the lagna is also rendered strong,
> **chances of success of the enterprise would be by far the greatest.**"

*Chances… by far the greatest* — a probability, not a promise. Exactly the correction David made:
*"These factors are intended to improve the probability of favorable outcomes, not ensure them."*
So a Siddha Yoga must **never clear a veto**, the same rule already enforced for Amrita Siddhi.

## ⚠ THREE THINGS I WILL NOT GUESS — David's call

The 1941 scan's OCR is dirty (it renders "Anuradha" as *Anuradlia*, "Rohini" as *Roliini*). Three
readings cannot be recovered from spelling alone, and each changes which real days fire:

1. **Bare "Uttara"** (Sunday, Tuesday, Wednesday, Friday) — Uttarashadha and Uttarabhadra are named
   separately in the same sentences, so this should be **Uttara Phalguni**. Near-certain, but it is
   an inference, not the page.
2. **"Blwvishta"** (Monday) — almost certainly **Sravishtha**, the classical alternate name for
   **Dhanishtha**. If so, Monday's list holds Dhanishtha.
3. **"Animidha"** (Wednesday) — almost certainly **Anuradha**, which the same page misspells
   *Anuradlia* twice.

Each is likely right. None is certain, and a wrong entry here silently mislabels a real day in a
real person's calendar forever. **Nothing from this grid is encoded until he rules**, which is the
same standard that kept me from typing it from memory in the first place.
