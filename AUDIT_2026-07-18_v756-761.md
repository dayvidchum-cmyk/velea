## Findings (4 — all fixed in v762)

1. **HIGH — one unveiled paint in the returning-login veil.** `bootVeilVisible` dropped on
   `firstRunState.data` ARRIVAL, but the splash mounts in the effect AFTER that render — one
   frame of Today between them, the exact flash class of the day. Fix: veil keyed on the
   `velea_splash` flag (consumed in the same commit the splash mounts). Safe because every
   beat renders above the veil (manifesto z-140, splash/welcome z-9999/130 vs veil z-125).
2. **MEDIUM — almanac year walk consulted the override tier.** Passing `profileId` meant a
   per-date override that lands exactly on yearStart relocated the ENTIRE year's almanac to
   that one day's city. The walk wants the stable place; profileId removed.
3. **MEDIUM — missing-location prompt gated on an unreachable flag.** `seen("welcome")` is
   never set for accounts whose welcome burned by show-cap (the same class v761 fixed for the
   greeting) — those accounts would never be prompted at all. Gate → `seen("manifesto")`.
4. **LOW — LocationSheet kept stale status across opens.** With reload-on-save gone the sheet
   persists; a prior error/success re-showed on the next open. Status/error reset on open.

  (input.lat client override still wins in Master Mode).
